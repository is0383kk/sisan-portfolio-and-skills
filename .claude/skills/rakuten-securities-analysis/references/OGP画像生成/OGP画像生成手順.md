# ポートフォリオ構成比カード → OGP画像(og.png) 生成手順

ダッシュボード（`https://is0383kk.github.io/sisan-portfolio-and-skills/`）の
**資産構成ドーナツカード**を、SNS シェア用の OGP 画像 `og.png` として書き出すための手順書。

## このタスクが扱うもの

ダッシュボード本体は React/SVG をブラウザ上で描画するため、画像化するには本来 HTTP サーバ＋
ヘッドレスブラウザが要る（CORS の都合で `file://` 不可）。この手順では**それを使わず**、
`data/holdings.json` を読んで `js/model.js` の `computeModel()` 相当（評価額・損益・構成比・
ドーナツ幾何・凡例）を Python 側で再計算し、Pillow でカード 1 枚を直接 PNG に描く。
ブラウザ／サーバ不要で完結する。

描画内容は実画面の `.pf-donut-card` に対応する:

- 中央：評価額合計・含み益・含み益率
- ドーナツ：3区分（国内株/米国株/投資信託）の構成比リング（セグメント間に 3px ギャップ）
- 凡例：区分ごとの評価額・構成比バー・比率%・評価損益

## 前提（依存）

- Python に **Pillow** が必要（`py -m pip install pillow`）。
- フォントは Windows 標準同梱の游ゴシック（`C:/Windows/Fonts/YuGothB.ttc` / `YuGothR.ttc`）を使う。

## 使い方（基本）

同梱の Python スクリプトを呼ぶだけでよい:

```
py "{SKILL_DIR}/scripts/make_og.py" [<holdings.json の絶対パス>] [--out <og.png の絶対パス>]
```

- 入力・出力とも引数で受け取り、`convert.py` / `nisa.py` と同じ流儀に揃えている。
- **引数を省略すると**リポジトリ直下の既定（`data/holdings.json` → `og.png`）にフォールバックする。
  通常はこの既定運用でよい（`py "{SKILL_DIR}/scripts/make_og.py"` だけで `og.png` が出る）。
- スクリプトは標準出力に**書き出し先のパスを 1 行**返す。実行後そのパスをユーザーに伝えること。

`{SKILL_DIR}` はこのスキル（SKILL.md）があるディレクトリ。

## 数値の注意点（重要）

- スクリプトの為替は holdings.json の `usdRateFallback`（固定値）を使う。実画面はライブ為替なので、
  **合計・米国株などの円換算額が数百〜千円ほどズレうる**。構成比（%）は一致する。
  再現性を担保するための割り切りであり、ロジックのズレではない。
- 数値ロジックを変えたいときは `data/holdings.json` の実データと `js/data.js` の `defaultConfig()` の
  両方を同期させる方針に従う（ダッシュボード側の前提）。

## 公開時の注意（プライバシー）

`og.png` には**実際の評価額・損益額がそのまま写る**。これを GitHub Pages 等で公開すると、
**誰でもアクセスできる URL で資産額が閲覧可能**になり、OGP 画像は SNS やクローラにキャッシュ・
インデックスされて後から消しても残りやすい。リポジトリの方針（個人の資産データはコミットしない）
とも矛盾しうるため、**公開・コミットはユーザーの明示的な判断を得てから**行うこと。
金額を伏せたい場合は、ダミー値の holdings.json を `--out` 付きで渡して別画像を作る運用ができる。

## やらないこと（このタスク固有）

- ドーナツ上の % ラベルは描かない（小さく読みづらいため。比率は右の凡例で示す）。
- 勝手に `index.html` へ OGP メタタグを追記したり、`og.png` を git に add・commit・push しない
  （公開判断はユーザーの領分）。
