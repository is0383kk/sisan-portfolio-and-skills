# 楽天証券 資産状況CSV → holdings.json 更新手順

楽天証券からダウンロードした資産残高CSVを基に、ダッシュボード（`index.html`）が読み込む
`data/holdings.json` を生成（更新）するための手順書。

## このタスクが扱うもの

`data/holdings.json` は保有銘柄の配列で、ダッシュボードの評価額集計・ドーナツ・損益ランキング
などの計算元になる。CSV の保有商品詳細から、この JSON の**数値項目**を最新化する。

重要なのは、CSV には JSON が必要とする一部の情報が無い/そのまま使えない点。よって
**「既存 `holdings.json` をキュレーション情報源として、CSV の数値だけ上書きするマージ方式」** を採る。

| JSON 項目 | 由来 | CSV 更新時の扱い |
| --- | --- | --- |
| `shares`（保有数量） | CSV「保有数量」 | **CSV で更新**（複数口座は合算） |
| `avg`（平均取得価額） | CSV「平均取得価額」 | **CSV で更新**（複数口座は数量加重平均） |
| `price`（現在値） | CSV「現在値」 | **CSV で更新** |
| `code`（短縮コード） | 手作業（例 "S&P500"/"オルカン"） | 既存値を維持（投信は CSV のコード列が空のため） |
| `name`（表示名） | 手作業で整形（末尾の別名を除去等） | 既存値を維持 |
| `monthAbs`（前月比） | 手作業 | 既存値を維持（**CSV には前日比しか無く前月比が無い**） |
| `cat`/`cur`/`unit`/`per` | 分類・単位 | 既存値を維持（新規追加時のみ CSV から推定） |
| `usdRateFallback` | CSV「参考為替レート／米ドル」 | CSV で更新 |

### 複数口座の合算

同一銘柄が複数口座（`NISA成長投資枠`／`NISAつみたて投資枠`／`つみたてNISA`／特定口座など）に
分かれている場合、ダッシュボードは銘柄単位で扱うため合算する。
- `shares` = 各口座の数量の合計
- `avg` = 数量で加重平均した取得単価（`Σ(shares×avg) / Σshares`）
- `price` = 現在値（各口座で同一）

### 突合キー（CSV ↔ 既存 JSON）

- 個別株：CSV ティッカー == 既存 `code`
- 投信：名称の完全一致 → 前方一致（同一 `cat` 内でのみ照合。投信はコード列が空のため名称で照合する）

## 使い方（基本）

CSV のパスがわかったら、同梱の Python スクリプトを呼ぶだけでよい:

```
py "{SKILL_DIR}/scripts/convert_holdings.py" "<入力CSVの絶対パス>"
```

- 既定では**リポジトリの `data/holdings.json` を直接上書き更新**する。
- 上書き前に結果を確認したい場合は `--out` で一時ファイルへ出力し、差分を確認してから反映する:
  ```
  py "{SKILL_DIR}/scripts/convert_holdings.py" "<CSV>" --out "<一時パス>.json"
  ```
- キュレーション情報源にする既存 JSON を明示するには `--holdings <絶対パス>` を付ける
  （既定はリポジトリの `data/holdings.json`）。
- スクリプトは標準エラーに差分の警告（除外/追加）を、標準出力に書き出し先パスを 1 行返す。

`{SKILL_DIR}` はこのスキル（SKILL.md）があるディレクトリ。

### 差分の読み方（重要）

スクリプトは黙って銘柄を切り落とさない。次を `[warn]` として列挙するので必ずユーザーに伝えること:

- **除外**：既存 JSON にあり CSV に無い銘柄。売却済みの可能性が高い。意図した売却なら除外でよい。
- **追加**：CSV にあり既存 JSON に無い銘柄。**投信は `code` が空・`monthAbs=0` で追加される**ため、
  追加後に `code`（短縮名）・`name`（表示名の整形）・`monthAbs`（前月比）を手動でキュレーションする。

### 起動コマンド・日付判定・文字化けの注意点（CSV変換と共通）

入力ファイル名まわりの落とし穴は CSV→Markdown 変換と同じ。
`references/CSV2マークダウン/CSV変換手順.md` の以下の節を共通の注意として参照すること:

- **「ファイル名に括弧を含む場合の注意」**：既定ファイル名 `assetbalance(all)_...csv` の `(all)` の丸括弧により、
  pyenv-win のバッチシム経由だと失敗する。起動は `py` ランチャーを第一候補にする。
- **「標準出力／標準エラーの表示が化けても、生成ファイルは正しい」**：コンソールが CP932 だと
  警告や出力パスの日本語が表示だけ化けることがある。生成される `holdings.json` は UTF-8 で正しい。
- **日付の判定ロジック**：`_comment` の「(yyyy/mm時点)」は入力ファイル名の 8 桁日付から決める。
  取れなければエラー停止（推測でフォールバックしない）。

## 出力後の確認

- `data/holdings.json` の内容は `js/data.js` の内蔵フォールバック（`defaultConfig`）と同じに保つ方針。
  乖離が問題になる場合は `js/data.js` 側も合わせて更新する（このスクリプトは触らない）。
- 反映を目視確認するときは簡易HTTPサーバでダッシュボードを開く（例: `python -m http.server`）。
  `file://` では `data/*.json` を fetch できない。

## やらないこと（このタスク固有）

- `monthAbs` を CSV の「前日比」で代用しない（前月比とは意味が異なる。既存値を維持する）。
- `code` や表示名を CSV の生の値で上書きしない（手作業キュレーションを尊重する）。
- 除外された銘柄を黙って消さない（必ず警告として提示し、売却かどうかの判断を仰ぐ）。
- `data/nisa.json` や `data/history.json` はこのスクリプトの対象外（別機能・別ソース）。
