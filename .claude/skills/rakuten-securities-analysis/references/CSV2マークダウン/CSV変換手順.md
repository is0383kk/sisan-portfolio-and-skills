# 楽天証券 資産状況CSV → Markdown 変換手順

楽天証券からダウンロードした資産残高CSVを、レポート用の Markdown に変換するための手順書。

## このタスクが扱うもの

楽天証券のサイトから出力できる資産残高CSV（文字コード CP932 / Shift-JIS）には、次の3つのセクションが順に並んでいる:

1. `■資産合計欄` … 資産合計、保有商品の評価額内訳、預り金、楽天銀行普通預金残高
2. `■ 保有商品詳細 (すべて）` … 個別銘柄の明細
3. `■参考為替レート` … 主要通貨の対円レート

これらをそのままの構成で Markdown の見出し＋表に整形する。
元データの数値・記号（`+` プレフィックス、`-`、`%` など）は変換せずそのまま転記し、表の桁数を勝手に丸めたり、項目を取捨選択したりしない。

## 使い方（基本）

CSV のパスがわかったら、同梱の Python スクリプトを呼ぶだけでよい:

```
py "{SKILL_DIR}/scripts/convert.py" "<入力CSVの絶対パス>"
```

- 出力先は **CSV と同じディレクトリ** に自動で `楽天証券_資産状況_yyyymmdd.md`（例: `楽天証券_資産状況_20260530.md`）として保存される。
- 出力先を明示したいときは `--out <絶対パス>` を付ける。
- スクリプトは標準出力に書き出し先のパスを 1 行返す。実行後にそのパスをユーザーに伝えること。

`{SKILL_DIR}` はこのスキル（SKILL.md）があるディレクトリ。
Claude Code 上では通常 `C:\Users\<ユーザー>\.claude\skills\rakuten-securities-analysis` または同等のパス。

### ファイル名に括弧を含む場合の注意

楽天証券の既定ファイル名は `assetbalance(all)_YYYYMMDD_HHMMSS.csv` で、**`(all)` の丸括弧** が含まれる。
Windows で `python` コマンドが pyenv-win のバッチシム（`python.bat`）に解決される環境では、このバッチが括弧を特殊文字として誤処理し、`... was unexpected at this time.`（終了コード 255）で失敗する。引数をクォートしても回避できない。

このため、起動コマンドは **バッチシムを経由しない `py` ランチャー** を第一候補にする。`py` が無い環境では `python.exe` の実体を直接指定する（例: `(python -c "import sys;print(sys.executable)")` で得たパス）。
素の `python` で括弧付きファイル名を渡すのは避ける。

なお `--out` で出力先を指定する場合、出力パス側に括弧が無ければ問題は入力パス側だけで起きる点に注意（入力に括弧があれば同様に `py` / `python.exe` 実体で起動する）。

### 標準出力のパス表示が化けても、生成ファイルは正しい

スクリプトは書き出し先のパスを標準出力に 1 行返す。Windows のコンソールが CP932 のままだと、この行の日本語部分が `楽天証券_資産状況_20260530.md` → `▒▒▒▒_20260530.md` のように **表示だけ** 化けることがある。これはターミナル表示の問題で、実際に生成されるファイル名・中身は UTF-8 で正しい。
ユーザーに出力先を伝えるときは、化けたコンソール表示をそのまま転記せず、`楽天証券_資産状況_yyyymmdd.md` という規則（または `--out` で指定した値）に基づいて伝えること。
確実を期すなら、書き出し後に出力ディレクトリを一覧して実ファイル名を確認するとよい。

## 日付の判定ロジック

出力ファイル名に使う `yyyymmdd`（年月日）は、**入力ファイル名に含まれる `YYYYMMDD` の 8 桁日付** から決定する。
楽天証券の既定ファイル名 `assetbalance(all)_YYYYMMDD_HHMMSS.csv` には必ずこの日付が含まれる。

8 桁日付が取れない場合はエラーで停止する（黙ってフォールバックしない）。これは、推測でズレた日付を付けないため。
為替レート時刻 `(MM/DD HH:MM)` には**年が無く日付を確定できない**ため、判定には使わない。

なお `assetbalance(all)_20260530_161938.csv` のように **5/30 に出力されたデータ** は `楽天証券_資産状況_20260530.md` として保存される。
ファイル名の日付と「ユーザーが何日時点として扱いたいか」がズレる場合は、`--out` で出力先を指定し直してもらう前提で運用する。
出力後にユーザーへ「{yyyymmdd} として保存しました」と明示すること。

## 出力フォーマット（厳守）

スクリプトはこの構造で出力する。手作業で書くときも同じ並びを維持すること。

```
# 楽天証券 資産状況（{年}年{月}月{日}日時点）

## 資産合計欄

| 項目 | 時価評価額[円] | 前日比[円] | 前日比[％] | 前月比[円] | 前月比[％] | 評価損益[円] | 評価損益[％] | 実現損益[円] | 配当・分配金[円貨] | 配当・分配金[外貨] |
| ... | ---: | ... |
| **資産合計** | **<金額>** | ... |

### 保有商品の評価額内訳
（同じ列構成で複数行）

### 預り金
| 区分 | 金額[円] | 前日比[円] | 前日比[％] | 前月比[円] | 前月比[％] |

### 楽天銀行普通預金残高
未取得（または値）

---

## 保有商品詳細（すべて）

| 種別 | コード・ティッカー | 銘柄 | 口座 | 保有数量 | 単位 | 平均取得価額 | 単位 | 現在値 | 単位 | 前日比 | 単位 | 時価評価額[円] | 時価評価額[外貨] | 評価損益[円] | 評価損益[％] |

---

## 参考為替レート

| 通貨 | レート | 単位 | 取得時刻 |
```

数値列は右寄せ（`---:`）、ラベル列は左寄せ。`資産合計` 行の項目名と金額のみ `**...**` で太字にする。
空セルは `-` で埋める（テーブルがガタつかないようにするための単なる見た目上の処理。元値が `-` の箇所もそのまま `-`）。

## やらないこと（変換タスク固有）

- 数値の丸め・通貨換算・損益率の再計算はしない。元 CSV の値をそのまま転記する。
- この変換の文脈では、銘柄ごとのコメントや投資アドバイスは生成しない。
