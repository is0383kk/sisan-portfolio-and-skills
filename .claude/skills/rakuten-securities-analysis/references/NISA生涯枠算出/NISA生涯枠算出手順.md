# 楽天証券 資産状況CSV → NISA生涯枠 算出手順

楽天証券からダウンロードした資産残高CSVを基に、新NISA の非課税枠（生涯）の使用状況を
算出し、`NISA生涯枠_yyyymmdd.md` というレポートに整形するための手順書。

## このタスクが扱うもの

新NISA の「非課税保有限度額（生涯投資枠）」は **1,800 万円**で、うち **成長投資枠は最大 1,200 万円**。
重要なのは、この枠が **簿価（取得価額）で管理される**点。値上がり益や評価損は枠の消費量を変えない。

楽天証券の資産残高CSV（`■ 保有商品詳細` セクション）には簿価の列が無いため、次式で復元する:

```
簿価（取得価額） = 時価評価額[円] - 評価損益[円]
```

評価益が出ていれば簿価は時価より小さく、評価損なら時価より大きくなる。これが各銘柄の「枠の使用額」になる。

どの枠に属するかは保有明細の **「口座」列**で判定する:

| 口座列の表記 | 扱い |
| --- | --- |
| `NISA成長投資枠` | 成長投資枠（生涯 1,200 万円の内枠）に算入 |
| `NISAつみたて投資枠` | つみたて投資枠に算入 |
| `つみたてNISA` | 旧つみたてNISA。新NISA の生涯枠とは**別制度**なので算入しない（参考として併記） |
| 上記以外（特定/一般口座、外貨預り金 など） | NISA 対象外として無視 |

## 使い方（基本）

CSV のパスがわかったら、同梱の Python スクリプトを呼ぶだけでよい:

```
py "{SKILL_DIR}/scripts/nisa.py" "<入力CSVの絶対パス>"
```

- 出力先は **CSV と同じディレクトリ** に自動で `NISA生涯枠_yyyymmdd.md`（例: `NISA生涯枠_20260628.md`）として保存される。
- 出力先を明示したいときは `--out <絶対パス>` を付ける。
- スクリプトは標準出力に書き出し先のパスを 1 行返す。実行後にそのパスをユーザーに伝えること。

`{SKILL_DIR}` はこのスキル（SKILL.md）があるディレクトリ。

### 起動コマンド・日付判定・文字化けの注意点（CSV変換と共通）

入力ファイル名・出力ファイル名まわりの落とし穴は CSV→Markdown 変換と同じなので、
`references/CSV2マークダウン/CSV変換手順.md` の以下の節を共通の注意として参照すること:

- **「ファイル名に括弧を含む場合の注意」**：既定ファイル名 `assetbalance(all)_...csv` の `(all)` の丸括弧により、
  pyenv-win のバッチシム経由だと失敗する。起動は `py` ランチャーを第一候補にする。
- **「標準出力のパス表示が化けても、生成ファイルは正しい」**：コンソールが CP932 だと出力パスの日本語が
  表示だけ化けることがある。生成物は UTF-8 で正しい。出力先は `NISA生涯枠_yyyymmdd.md` の規則に基づいて伝える。
- **日付の判定ロジック**：出力名の `yyyymmdd` は入力ファイル名の 8 桁日付から決める。取れなければエラー停止（推測でフォールバックしない）。

## 出力フォーマット

スクリプトはこの構造で出力する。手作業で書くときも同じ並びを維持すること。

```
# NISA非課税枠（生涯）使用状況（{年}年{月}月{日}日時点）

（簿価で管理される旨と復元式の説明文）

## 枠サマリー

| 枠 | 非課税保有限度額[円] | 使用額（簿価）[円] | 残枠[円] | 使用率 |
| ... | ---: | ---: | ---: | ---: |
| **生涯投資枠（合計）** | **18,000,000** | **<使用額>** | **<残枠>** | **<使用率>** |
| └ 成長投資枠 | 12,000,000 | ... |
| └ つみたて投資枠 | 単独上限なし | <使用額> | — | — |

（枠の上限・旧つみたてNISAの扱いに関する注記）

## 参考：旧つみたてNISA（生涯枠の対象外）
（旧つみたてNISA口座の銘柄があれば、銘柄ごとの簿価と合計を表で併記。無ければ省略）
```

- 「使用率」は `使用額 / 上限 × 100`（小数第1位）。つみたて投資枠には単独の生涯上限が無いため上限・残枠・使用率は `—`。
- 旧つみたてNISA の保有が無いCSVでは「参考」セクションごと省略してよい。

## やらないこと（このタスク固有）

- 時価ベースで枠の使用額を出さない（必ず簿価で集計する。これを誤ると枠が過大に見える）。
- 旧つみたてNISA を生涯枠の使用額に合算しない（別制度のため）。
- 銘柄ごとの投資アドバイスや今後の枠の使い方の提案は、明示的に求められない限り生成しない。
